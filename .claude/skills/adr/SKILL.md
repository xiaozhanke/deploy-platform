# adr (Architecture Decision Record)

为一个架构 / 设计决策创建新的 ADR 文档，沉淀到 `docs/adr/`。

## 参数

`$ARGUMENTS` = 本次决策的主题简述（可选）。为空时先问用户这条 ADR 要记录什么决策，不要凭空编造。

## 步骤

1. `ls docs/adr/` 找出当前最大编号，新编号 = 最大编号 + 1，补零到 4 位（例如已有到 `0018` 则新建 `0019`）。
2. 与用户对齐以下要素，信息不足就追问，**不要替用户拍板技术选型**：
   - 决策标题（简体中文）与英文副标题
   - 背景与动机（为什么现在要做这个决策）
   - 候选方案，含**被否决的方案及其否决理由**
   - 最终选定的方案
   - 决策带来的后果（含对既有 ADR 的影响）
3. 文件名用 `NNNN-kebab-case-english-title.md`，写入 `docs/adr/`。
4. 套用仓库既有 ADR 的结构（参照同目录已有文件）：
   - 一级标题：`# 简体中文标题 (English Title)`
   - 紧跟一段背景与决策综述
   - `## Considered Options`：逐条列候选方案，每条说明取舍；被否决项写明「否决」及原因，选定项标注「选定」
   - `## Consequences`：列出该决策的正负后果；若反转 / 修订了此前某篇 ADR 的结论，显式点明是哪一篇
5. 全文用简体中文撰写。写完后回报新文件的路径与编号。
